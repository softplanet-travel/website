# SoftPlanet MVP 功能驗收紀錄

驗收日期：2026-07-13  
驗收環境：本機 HTTP Server、Chromium、訪客模式

> 此文件保存當時的 MVP 驗收紀錄。專屬小卡的 Google／Naver 專用欄位與雙導航描述，已由 `docs/softplanet-v1-design-system.md` 的通用參考連結及單一「查看地圖」規則取代。

## 核心流程結果

| 流程 | 實際驗證結果 |
|---|---|
| A. 記帳與預算 | 東京旅行新增 JPY 1,200 餐飲支出，依示範匯率換算 TWD 252；設定 TWD 30,000 預算後顯示已用 1%、剩餘 29,748。 |
| B. 正式地點導航 | 搜尋東京晴空塔後進入詳情；查看地圖與開始導航皆使用安全的 Google Maps 分享連結。 |
| C. 日本自訂卡片 | 建立「新宿小花園」，從 Area Master 選擇新宿，附 Google Maps 分享連結並加入東京旅行。 |
| D. 韓國自訂卡片 | 建立「延南小公園」，從 Area Master 選擇弘大／延南；導航優先使用 Naver Maps，Google Maps 作為備援。 |
| E. 自訂卡片升級 | 自訂「東京 晴空塔」比對正式卡片後由使用者確認升級；保留旅行與備註，正式名稱、內容與地圖資料取代自訂欄位。 |
| F. 分區搜尋 | 收藏、旅行、旅行內地點、我的攻略與我的工具皆只搜尋各自範圍；「退稅」只顯示日本退稅攻略。 |
| 積木隱藏／恢復 | 隱藏旅行記帳後重新顯示，既有「午餐拉麵」與 JPY 1,200 支出仍保留。 |
| 輸入限制 | 自訂卡片備註輸入 101 字時顯示「超出 1 個字元，請縮短後再儲存」。 |

## Responsive 結果

首頁、旅行工作區、旅行記帳均在下列寬度以實際瀏覽器 viewport 驗證；所有尺寸都沒有頁面水平捲動、固定 Header／Bottom Navigation 裁切或控制項超出畫面。

| 寬度 | 結果 | 跑版 | CTA／導覽 |
|---:|---|---|---|
| 320px | 正常 | 無 | 正常 |
| 360px | 正常 | 無 | 正常 |
| 375px | 正常 | 無 | 正常 |
| 390px | 正常 | 無 | 正常 |
| 414px | 正常 | 無 | 正常 |
| 768px | 正常 | 無 | 正常 |
| 1024px | 正常 | 無 | 正常 |
| 1280px Desktop | 正常 | 無 | 正常 |

自訂卡片完整表單另在 320px 與 1280px 驗證：7 個控制項均在可視寬度內，沒有水平跑版。
